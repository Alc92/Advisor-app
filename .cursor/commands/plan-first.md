Usa la skill `implement-use-case`, pero no escribas código todavía.

Primero devuelve solo un plan breve con:

- objetivo funcional
- módulo correcto
- capas afectadas
- archivos a crear o tocar
- puertos necesarios
- tipo de tests
- riesgos

Reglas:

- no implementes aún
- no inventes arquitectura nueva
- elige el cambio más pequeño compatible con el objetivo

## Secuencias de prompts

Si el plan se va a dividir en varios prompts, cada prompt propuesto debe ser autocontenido.

No uses pasos que dependan de memoria implícita de prompts anteriores. Cada paso debe incluir:
- contexto mínimo
- objetivo del paso
- fuentes aplicables
- no debe romper
- puede tocar / no tocar
- criterios de aceptación
- testing esperado
